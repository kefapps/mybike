# MYB-63 - Unity HUD Difficulty, Slope And Current Segment

Status: implemented
Linear: https://linear.app/kefjbo/issue/MYB-63/myb-037-refondre-le-hud-unity-difficulte-pente-et-segment-courant
Branch: `myb-63-hud-difficulty-slope-segment`
Date: 2026-06-12

## Scope

Refresh the Unity HUD in `unity/Echapee4D` so the ride cockpit shows:

- existing speed, distance, progress and mock session state;
- effort / difficulty;
- current slope;
- current route segment.

The scope stays Unity-only. React/Three.js HUD work and hardware-bike pairing
remain out of scope.

## Implementation Notes

- Extended `MYB89ProbeRide` with `difficultyLabel`, `gradeLabel` and
  `segmentLabel`.
- Derived HUD route context from existing Unity route markers:
  - grade is computed from the current marker segment elevation delta;
  - segment labels are stable route bands: `Warmup`, `Climb`, `Sprint`,
    `Recovery`;
  - effort labels remain readable even when the route grade is subtle.
- Updated `MYB89ProbeBuilder` to create and wire the new HUD labels.
- Updated MYB-89 and MYB-91 validators so missing HUD wires or missing
  difficulty/slope/segment text fail validation.

## Review Notes

- The first preview capture showed a `Warmup` segment with `Sustained climb`
  effort because a tiny positive grade overrode the segment band.
- The difficulty label priority was adjusted so warmup and recovery bands stay
  semantically stable while the climb and sprint sections still read as effort.
- MYB95 texture importer metadata and MYB89 material float-normalization churn
  generated by Unity refresh were discarded as out-of-scope serialization noise.
