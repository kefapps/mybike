## Grill With Docs Decisions

- Canonical term: `Route Difficulty Cue` / indice visuel de difficulte.
- Style: sober diegetic fantasy.
- Visual treatment: emissive materials with a light bounded pulse animation.
- Placement: one entry marker/gate plus secondary beacons inside the segment.
- Covered segments: `Climb`, `Sprint`, `Recovery`.
- Excluded segment: no dedicated `Warmup` cue.
- First implementation uses Unity primitives generated by code.
- No Meshy, no third-party asset import, no hardware bike, no Three.js work.

## Implementation Notes

- Added `MYB48RouteDifficultyCueController`.
- Wired the controller from `MYB89ProbeBuilder`.
- Generated route-side cue primitives from route markers:
  - climb: warm orange emissive cue band;
  - sprint: stronger red emissive cue band;
  - recovery: calmer blue emissive cue band.
- Added entry glow lights and runtime pulse animation.
- Added `MYB48RouteDifficultyCueValidator`.
- Extended MYB-89 and MYB-91 validation reports with route difficulty cue counts.
